# 4. 디버깅 보고서 (10점)

> 💡 **디버깅이란?** 프로그램에서 발생한 오류(버그)를 찾아내고 고치는 과정입니다.

---

## 📝 작성 정보
- **작성 날짜:** 2025년 11월 20일
- **작성자:** 신은찬

---

## 🐛 발견한 오류 #1

### 1) 어떤 문제가 발생했나요?
컴파일이 안 돼요

### 2) 오류가 발생한 코드
```c
// return 0
```

### 3) 왜 오류가 생겼나요?
세미콜론(;)을 빼먹었어요 

### 4) 어떻게 고쳤나요?
```c
return 0;
```
세미콜론을 다시 붙임
### 5) 고친 후 결과
- ✅ 잘 작동해요!

## 🧪 프로그램 테스트 결과

### 테스트 1
- **무엇을 테스트했나요?** 도서 정보 입력
- **어떻게 테스트했나요?** 도서 이름에 "홍길동전", 저자에 미상을 입력했어요
- **결과:** ✅ 성공 


---

## 💭 디버깅하면서 배운 점

- 세미콜론(;)을 작성하지 않으면 컴파일 과정에서 오류가 발생한다는 것을 배웠어요.


---

## ✅ 최종 체크리스트

프로그램이 잘 작동하는지 최종 확인해보세요!

- [O] 컴파일이 잘 되나요? (오류 메시지가 없나요?)
- [O] 프로그램이 실행되나요?
- [O] 메뉴가 제대로 나오나요?
- [O] 입력한 값이 정상적으로 처리되나요?
- [O] 모든 기능이 잘 작동하나요?
